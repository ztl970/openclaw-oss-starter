# 日常工作交接清单

## 目标

把适合长期交给 `main` 承接的日常工作明确下来，避免只交“发布”，不交“日常运行”。

## 可交接的日常工作

### 1. 日常同步检查

由 `main` 执行：

- 检查 GitHub 是否有需要同步的说明或展示更新
- 检查 ClawHub 版本或描述是否需要同步
- 检查飞书知识库页面是否需要更新
- 检查对外文案是否需要刷新

### 2. 日常任务承接

由 `main` 作为正式群的任务入口：

- GitHub 项目处理
- 新 skill 上线流程
- 知识库同步
- 发布前检查
- 文案更新请求

### 3. 日常沉淀

由 `main` 推动沉淀：

- 更新通用指令库
- 更新发布记录与回执
- 更新后续待办
- 更新规则页或复盘页

### 4. 日常分流

由 `main` 判断：

- 本地代码任务
- 公开同步任务
- 知识库沉淀任务
- 混合任务

需要公开同步时，再派给 `media`。

## 交接顺序

### 第一阶段

先交：

- 日常同步检查
- 知识库同步
- GitHub 项目处理入口

### 第二阶段

再交：

- 新 skill 上线流程
- 发布前检查
- 文案更新与最小同步动作

### 第三阶段

稳定后再交：

- 通用指令沉淀
- 周度复盘推动
- 更完整的发布节奏管理

## 日常触发方式

### 手动

在正式群发短指令：

- 请从 main 做一次日常同步检查。
- 请从 main 同步 OpenClaw 开源项目知识库。
- 请从 main 处理这次 GitHub 项目工作。

### 自动

通过自动化执行：

- Daily Check
- Weekly Review
- Command Digest

## 交接完成的标志

- `main` 能稳定接住日常项目任务
- `main` 能稳定给出标准回报
- `main` 能把公开同步任务派给 `media`
- `小果` 能稳定完成知识库与公开同步

## 一句话规则

先把“检查、承接、分流、沉淀”这四类日常交给 `main`，再逐步扩大范围。
