# 日常工作边界表

## 目标

明确哪些日常工作可以交给 `main`，哪些需要你保留最终拍板，哪些需要 `果爸` 执行。

## 边界矩阵

| 工作类型 | 是否可交给 `main` | 是否需派给 `media` | 是否需你拍板 | 是否需 `果爸` 执行 |
| --- | --- | --- | --- | --- |
| 日常同步检查 | 是 | 如需公开同步则是 | 否 | 否 |
| GitHub 项目说明更新 | 是 | 如需公开同步则是 | 通常否 | 否 |
| 知识库规则页更新 | 是 | 是 | 通常否 | 否 |
| 发布前检查 | 是 | 否 | 通常否 | 否 |
| skill 分级判断 | 是 | 否 | 重要边界时建议是 | 否 |
| 新 skill 上线流程 | 是 | 是 | 方向变化时建议是 | 否 |
| ClawHub 公开同步 | 是 | 是 | 重要发布时建议是 | 否 |
| 对外文案整理 | 是 | 是 | 重要表达时建议是 | 否 |
| 小红书实际发布 | 否 | 可协助准备文案 | 是 | 是 |
| 项目方向调整 | 否 | 否 | 是 | 否 |
| 是否公开某项敏感内容 | 否 | 否 | 是 | 否 |
| 私有内容是否抽象后再发 | 可先给建议 | 否 | 是 | 否 |

## 推荐理解

### 可以放心交给 `main`

- 日常检查
- 日常承接
- 日常分流
- 日常沉淀
- 基于现有规则的标准判断

### 建议你保留拍板

- 项目方向变化
- 是否公开边界模糊的内容
- 是否将个人场景抽象成公开 skill
- 重要发布的最终口径

### 必须由 `果爸` 执行

- 国内平台账号实际发布
- 尤其是小红书账号侧动作

### `media / 小果` 最适合承接的

- 知识库同步
- 对外文案整理
- GitHub / ClawHub / 飞书公开同步
- 发布回执整理

## 一句话规则

标准化、可复用、可按规则判断的日常工作交给 `main`；涉及方向、隐私边界、国内账号最终发布的，仍由你或 `果爸` 拍板执行。
