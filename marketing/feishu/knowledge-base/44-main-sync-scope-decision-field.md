# 44_Main 的同步范围判断字段

## 目标

让 `main` 的标准回报里固定出现一项：

- `同步范围判断`

这样你一眼就能看出：

- 这次只更 GitHub
- GitHub + 飞书知识库
- GitHub + ClawHub
- 三者都要

## 推荐写法

回报结构建议改成：

```text
状态:
动作:
同步范围判断:
证据:
下一步:
```

## 同步范围判断示例

- `仅 GitHub`
- `GitHub + 飞书知识库`
- `GitHub + ClawHub`
- `GitHub + ClawHub + 飞书知识库`
- `仅本地保留，不进入公开链路`

## 推荐输出句式

```text
同步范围判断: GitHub + 飞书知识库
```

或：

```text
同步范围判断: 仅 GitHub，本次不需要更新 ClawHub 与飞书知识库
```

## 原则

- 先给结论
- 再在动作或证据里解释原因
- 不要把平台判断藏在长段落里
