# 50_多项目回报模板

## 目标

让 `main` 在多项目日常检查时，不再混成一段总述，而是按项目分段回报。

## 推荐结构

```text
状态: done
动作: 已完成多项目日常检查

openclaw-oss-starter
- 同步范围判断:
- 动作:
- 证据:
- 下一步:

言町驾驶舱
- 同步范围判断:
- 动作:
- 证据:
- 下一步:

羲果陪伴
- 同步范围判断:
- 动作:
- 证据:
- 下一步:
```

## 推荐同步范围判断

每个项目都应单独判断：

- `GitHub + 飞书知识库`
- `仅 GitHub`
- `仅内部沉淀`
- `仅本地保留`
- `商业化敏感，当前不进入公开链路`

## 原则

- 不要把不同项目混在一段话里
- 公开项目与商业化敏感项目必须分开写
- 有动作写动作，无动作也要写结论
