# 52_禁止全局扫描主目录规则

## 目的

阻止 `main` 在正式中枢群里执行这种高噪声、低稳定性的策略：

- 扫描主目录
- 枚举所有 Git 仓库
- 用仓库名猜目标项目

## 为什么不适合

- 容易扫到无关目录
- 容易遇到系统目录或权限问题
- 容易把 repo 名和本地目录名混淆
- 会让简单任务变成路径定位问题

## 正式规则

在正式中枢群中：

- 不允许把“扫描主目录找项目”作为默认第一步
- 不允许在没有必要时回退到全局仓库扫描

## 允许的第一步

优先顺序应为：

1. 读取默认项目上下文
2. 读取项目登记表
3. 只检查登记在册的项目路径
4. 如某路径失效，再回报最小缺口

## 只有这些情况才允许扩大扫描

- 你明确要求重新发现项目
- 项目登记表缺失且你明确允许扫描
- 已登记路径全部失效，且你要求自动修复

## 推荐回报

如果缺少登记表或真实路径，应回：

```text
状态: blocker
动作: 已完成规则检查，但当前缺少可用项目登记表
同步范围判断: 待判断，当前先不进入公开链路
证据: 默认上下文存在，但多项目路径登记缺失
下一步: 请补项目登记表或补充目标项目真实路径
```

## 一句话规则

在正式中枢群，`main` 默认应读登记表，不应扫主目录。
