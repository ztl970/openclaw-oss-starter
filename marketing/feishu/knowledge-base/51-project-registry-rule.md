# 51_项目登记表规则

## 问题

如果 `main` 没有项目登记表，它就容易退回到错误策略：

- 扫描主目录
- 枚举所有 Git 仓库
- 再猜哪个项目和当前任务有关

这在正式中枢群里既不稳，也容易误判。

## 正确做法

`main` 应先读取项目登记表，再开始多项目日常检查。

项目登记表至少应包含：

- 项目标识
- 本地路径
- GitHub 仓库
- 敏感度
- 是否允许公开同步

## 当前建议文件

使用：

- `config/main-project-registry.example.yaml`

后续你可以复制一份私有版，再把真实路径填进去。

## 默认规则

1. 先读项目登记表
2. 只检查登记表中的项目
3. 不因为“想更全面”而扫描整个主目录
4. 如果某项目未登记，再回报缺口，而不是全盘扫描

## 一句话规则

多项目检查应基于“项目登记表”，不是基于“扫主目录找仓库”。
