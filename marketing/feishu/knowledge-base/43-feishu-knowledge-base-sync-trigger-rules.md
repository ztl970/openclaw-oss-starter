# 43_飞书知识库同步触发规则

## 目的

让 `main` 先判断：

- 这次是否只需要 GitHub 更新
- 还是还要同步飞书知识库

## 核心原则

飞书知识库不是每次都必须跟着更新。

但只要变化已经影响：

- 项目总览
- 对外发布文案
- 申请素材
- 规则页
- 复盘页
- 协作说明

就应该把飞书知识库视为需要同步的公开面。

## 建议触发场景

以下情况默认应判断“需要同步飞书知识库”：

- README 首页定位明显变化
- skill collection 有新增或删除
- skill 版本状态变化需要沉淀
- 发布流程、职责边界、规则体系有更新
- 对外文案、申请文案、群协作文案有更新
- 项目阶段里程碑、复盘、待办有更新

## 通常不必同步的场景

以下情况通常只更新 GitHub 即可：

- 小幅排版优化
- 不影响公开理解的文字微调
- 仅代码层修补，但不影响对外展示
- 仅本地脚本实验
- 仍处于本地保留级别的内容调整

## 推荐判断顺序

1. 这次变化是否会影响公开理解
2. 这次变化是否会影响知识库中的总览、规则、文案或复盘
3. 如果会影响，就同步飞书知识库
4. 如果只是仓库内微调，可只更新 GitHub

## 一句话规则

- 影响公开说明与协作规则，就同步飞书知识库
- 仅仓库内微调，不必强制同步飞书知识库
