# 可发布 Skill 候选池

## 目的

这一页用于整理“适合继续做成公开 skill”的候选方向。

原则：

- 优先通用
- 优先 public-safe
- 优先能被别人复用
- 优先已经在本地工作流里验证过基本价值

## 当前已发布方向

- `family-homework-pomodoro`
- `daily-task-checkin`
- `practice-session-checkin`
- `mac-multi-instance-deployment`

这些方向已经证明：

- 可抽象成模板
- 可公开整理
- 可同步到 GitHub / ClawHub / 飞书知识库

## 适合继续扩展的候选方向

### 1. 通用协同类

- 任务 intake 与 check-in
- 提醒与完成确认
- 审批型工作流
- 路由与分流规则

### 2. 发布与运营类

- 项目发布公告
- 多平台文案改写
- 社区回复辅助
- 指令沉淀与发布检查

### 3. 知识库与归档类

- 项目知识库同步
- 阶段成果归档
- 发布回执记录
- 通用规则整理

### 4. 本地工作流与部署类

- 多实例部署
- 本地角色边界
- 公私内容分离
- 本地工具链检查

## 候选池筛选标准

一个候选 skill 越满足下面这些条件，越值得继续做：

- 能清楚描述“解决什么问题”
- 能写出明确边界
- 能写出 public-safe 示例
- 不依赖私人账号和私人环境
- 能形成标准文件结构

## 暂不优先的方向

- 强依赖个人身份的 skill
- 强依赖家庭内部细节的 skill
- 需要私密数据才能运行的 skill
- 只能用于某一个私人账号的 skill

## 建议流程

1. 先放进候选池
2. 由 `main` 做通用性与 public-safe 判断
3. 通过后再进入 skill 编写与发布流程
