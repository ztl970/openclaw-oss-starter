# 发布节奏与复盘

## 发布节奏建议

### 日常节奏

- 有明确新进展时再发，不为发而发
- 优先同步：
  - 新 skill
  - 新版本
  - 新发布渠道
  - 新文档沉淀

### 一次标准发布链路

1. 在仓库完成内容更新
2. 同步 GitHub
3. 如适合，发布到 ClawHub
4. 在飞书知识库沉淀更新内容
5. 由 `media` 生成对外发布文案
6. 由对应执行人完成平台发布

## 当前分工对应的发布节奏

### media

- 负责生成和整理对外文案
- 负责 Feishu 内部同步
- 负责把项目信息沉淀为可复用页面

### 果爸

- 负责小红书等国内平台实际发布
- 参考知识库里的对外发布文案做适配

### 小果

- 负责知识库页面更新、补充和归档
- 让项目资料长期可查、可复用

## 复盘重点

每次发布后建议记录：

- 发了什么
- 发到哪些平台
- 哪一版文案效果更好
- 是否带来新的互动或线索
- 下次应该保留什么、删掉什么

- 发布平台：
- 互动情况：
- 后续动作：
