# Contributing

Thanks for helping improve `openclaw-oss-starter`.

## What to contribute

- clearer docs
- safer automation scripts
- boundary/routing improvements
- public-safe examples

## Before you open a PR

- do not include secrets, tokens, private memory files, or heartbeat state
- keep changes focused
- update the README if the workflow changes

## Suggested workflow

1. fork or branch
2. make a small change
3. test the shell script or docs you touched
4. open a pull request with a short summary
